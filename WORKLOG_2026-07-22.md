## 알아둘 점

- 경제법 챕터 확장은 항상 `git pull origin master --no-edit`으로 원격 변경사항을 먼저 받은 뒤 커밋·푸시하는 방식으로 진행함 (다른 PC에서 동시 작업 중이었기 때문).
- 챕터 신규/확장 작업 패턴: ①`grep -c "category: '챕터명'"`으로 현재 문제 수 확인 → ②적절한 삽입 위치(같은 대분류 섹션 끝, `// N위:` 주석 앞) 파악 → ③세부유형(subTopic) 5개 안팎으로 나눠 6문제씩 작성 → ④`grep -c`로 목표 개수 재확인 + `grep -o "id: '[^']*'" | sort | uniq -d`로 중복 id 없는지 확인 → ⑤`flutter analyze`+`flutter test` 통과 확인 → ⑥`git pull`→`commit`→`push`.
- 민법·경영학은 여전히 기출 분석 미완료 상태(`topic_stats.dart`에서 questionCount 전부 0).
